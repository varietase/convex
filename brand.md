# Brand seed — X-Ray for AI Code

## Working identity
- **Public name:** X-Ray for AI Code
- **Short name:** X-Ray
- **Category phrase:** comprehension control layer for code
- **Promise:** See how your code fits together. Prove you understand it.
- **Anti-positioning:** not an IDE, not a code generator, not a chatbot that paraphrases a snippet.

## Voice
Direct, calm, and evidence-first. Explain unfamiliar concepts without talking down to the builder. Prefer “We found this call at `file:line`” over “The AI thinks…”. Admit uncertainty and unsupported relationships visibly.

## Visual direction
A dark technical canvas with warm “learned” states, cool structural edges, and restrained depth effects. The interaction metaphor is an optical instrument: focus, zoom, trace, reveal—not sci-fi scanning or medical diagnosis.

## UI copy rules
- Say **evidence**, **connection**, **concept**, **try explaining**, and **not enough evidence**.
- Do not say **magic**, **fully understands**, **mastered**, **guaranteed**, or **AI knows your code**.
- Every structural claim must expose its source location.
- Feedback critiques the explanation, never the learner.

## Accessibility baseline
WCAG 2.2 AA target, complete keyboard graph traversal, visible focus, non-color status indicators, reduced-motion support, and text alternatives for every graph path.
