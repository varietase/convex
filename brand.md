# Brand seed — convex

## Working identity
- **Public name:** convex
- **Short name:** convex (same as public name; use lowercase in prose)
- **Category phrase:** comprehension control layer for code
- **Promise:** See how your code fits together. Prove you understand it.
- **Anti-positioning:** not an IDE, not a code generator, not a chatbot that paraphrases a snippet.

## Voice
Direct, calm, and evidence-first. Explain unfamiliar concepts without talking down to the builder. Prefer “We found this call at `file:line`” over “The AI thinks…”. Admit uncertainty and unsupported relationships visibly.

## Visual direction
A premium, dark-first editorial instrument for code comprehension. The base canvas is near-black, typography is off-white, and neutral grays carry most surfaces, borders, dividers, and metadata. Warm coral/orange is the restrained accent for action, focus, selection, and meaningful learning-state changes. The interaction metaphor remains an optical instrument: focus, zoom, trace, reveal -- never sci-fi scanning, medical diagnosis, or playful gamification.

Use large, quiet compositions with generous whitespace, rounded outlined cards, thin borders, minimal shadows, subtle circular geometry, and smooth functional motion. Decorative gradients are not part of the core language. Visual polish must never obscure evidence, sample labels, source locations, or uncertainty.

## UI copy rules
- Say **evidence**, **connection**, **concept**, **try explaining**, and **not enough evidence**.
- Do not say **magic**, **fully understands**, **mastered**, **guaranteed**, or **AI knows your code**.
- Every structural claim must expose its source location.
- Feedback critiques the explanation, never the learner.

## Visual identity summary
- **Canvas:** near-black (`#070707`) with layered neutral surfaces.
- **Type:** editorial display headings paired with a clean sans-serif body and monospace code.
- **Accent:** warm coral (`#ff7a59`) used sparingly for primary action, focus, and selected evidence.
- **Components:** rounded cards, thin outlines, soft hover states, minimal elevation, and stable dense rails.
- **Icons:** thin-line, rounded-stroke icons with accessible names and text when meaning is not obvious.
- **Motion:** smooth, short transitions for continuity only; no fake progress or decorative animation.

## Accessibility baseline
WCAG 2.2 AA target, complete keyboard graph traversal, visible focus, non-color status indicators, reduced-motion support, text alternatives for every graph path, and contrast checks for all semantic token pairs in `docs/design-system.md`.
